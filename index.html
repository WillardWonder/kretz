Kretz AI Decision Guide

A governance-first decision tool. Users describe a problem, then the GPT helps decide whether it should be AI, existing software, automation, a narrow GPT, process cleanup, vendor system, or no AI for now.

Open GPTUse in activity

Hands-on activity: try the tools, then decide what to build

  1. Start with the Kretz AI Decision Guide. Describe a real Kretz problem or workflow idea.
  2. Answer its questions and see whether it recommends existing software, automation, a Custom GPT, process cleanup, a vendor system, or no AI for now.
  3. If the problem is communication-related, try the Kretz DNA Assistant with a rough email or message.
  4. Discuss what data the tool needs, what it should never do, who owns it, and what human review is required.

Access note: participants can use shared GPTs if they are signed in and have access to the GPT. Creating or editing GPTs requires a paid subscription and is done in the web experience. If someone cannot access a GPT during the session, they can still complete the activity by pasting the short instructions into a normal ChatGPT conversation.

What Kretz does next.

Move from excitement to action with a governance-first roadmap.

Refine AI policyClarify allowed uses, protected data, approved tools, and review requirements.
Select secure toolsEvaluate ChatGPT Team, Copilot, and other platforms based on usability, controls, cost, and longevity.
Create the Sign-Off RuleNo external, customer-facing, financial, safety, or technical AI-assisted content goes out without human review.
Use No-Fault ReportingMake it easy to report hallucinations, errors, and tool problems so processes improve.
Define Kretz DNACapture voice, tone, values, vocabulary, and owner-thinker culture for AI-assisted work.
Name internal ownersDefine leadership sponsor, IT/security contact, department champions, and potential tool builder role.
