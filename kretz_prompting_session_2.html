Two hours · hands on the keyboard

Prompting with purpose, not generic AI output.

The software is available to everyone. The difference comes from how clearly you define the purpose, audience, context, constraints, evidence, and review process.

AI fills in what you leave blank.

Generic input produces the statistical middle: competent-looking work that sounds and looks like everyone else. Intentional prompts make the output more useful, more honest, and more recognizable as Kretz work.

Never paste sensitive company information into an unapproved AI tool.
Practice uses synthetic files so this rule never has to pause for training.

Eight hands-on activities.

Run them in order. The first three teach specificity and intention. The next four apply the same skill to data and daily work. The last activity turns a description into a working tool.

Bowl of fruit

Everyone runs the same prompt and compares screens. Similar output is the lesson.

Run together
Create an image of a bowl of fruit.
Teaching point: vague input produces the average. The average carries none of your language, judgment, or style.

House in Wisconsin

Start generic, then add the details that remove entire categories of wrong answers.

Generic
Create an image of a house in Wisconsin.
Structured
Create a realistic image of a house in Wisconsin using: house style, exterior color, roof, setting, season, time of day, yard details, and purpose. Do not include address, license plates, mailbox numbers, or identifying personal details.
Correction loop
Keep everything exactly as it is, but change the siding from white to tan, remove the porch swing, and move the garage to the left side of the house.

Intention changes output

Use the same house twice. Only the reason changes.

For a child
Transform the same house into a black-and-white coloring page for a child.
For a remodel conversation
Transform the same house into an architectural concept image for a remodel conversation.
The word “for” does real work. Say who will use the result and what happens next.

Spreadsheets by prompt

Use the haul-cost CSV. Ask for structure, formulas, assumptions, and warnings—not a magical answer.

Build the plan
Create a spreadsheet plan for tracking haul costs. Include columns, sample rows, formulas, cost per mile, cost per MBF, assumptions, and review warnings.
Ask for formulas
Now give me the actual Excel formulas in the cells — do not give me the calculated numbers. Flag anything you had to assume.
Teach me back
Looking back at everything we just built together, how could I have gotten all of this from a single prompt? Write me that prompt, then explain which parts of it were doing the work.
AI is not a calculator. Use it to propose structure and formulas. Make it show its work, let Excel perform the arithmetic, and verify the result.

Landowner research boundaries

Research where public records live, verify the source, then practice on the synthetic CSV.

Shot 1 — find the data
How can I find all the landowners in a single Wisconsin county using free, public data? I want owner names, mailing addresses, and acreage. Tell me exactly where that data lives, who publishes it, what format it comes in, what it costs, and how current it is.
Shot 2 — narrow it
Now narrow it down. I only want private forested parcels big enough to be worth a conversation about timber. In Wisconsin parcel data, which field tells me a parcel is forest land, and what are the actual class codes? What acreage threshold makes sense, and why? Also explain how Managed Forest Law enrollment works and what its minimum acreage requirements are.
Shot 3 — forestry layer
Where do I get Managed Forest Law spatial data for Wisconsin, and how would I join it to the county parcel data? Give me the portal, the layer name, and the field I would join on. Tell me plainly what parts of MFL are published publicly and what parts are not.
Synthetic practice
Using the synthetic landowner research CSV, create a verification checklist and a careful outreach draft. Do not estimate timber value, imply Kretz inspected the property, or treat public data as verified.
Public does not mean anything goes. Verify current county records, follow approved outreach practices, avoid protected or suppressed records, and require human review.

Operational efficiency

Not every problem needs AI. Classify first, then add real-world constraints.

Classify
Using the operational efficiency CSV, classify each task as existing software first, narrow Custom GPT, automation, process cleanup, vendor system, or do not use AI yet.
Add constraints
Now redo the top three with my actual constraints: who is on shift, what equipment is down, and what has a hard deadline this week. If a constraint makes your recommendation wrong, say so instead of working around it.

Price movement scenarios

The synthetic data contains a planted pattern and deliberate red herrings.

Analyze
Using the synthetic lumber price movement data, identify patterns and possible leading indicators. Do not make a prediction as fact. Give scenarios, assumptions, confidence, and source-verification needs.
Build the monitoring template
Now turn that into a one-page monitoring template: the drivers worth tracking, where each number comes from, how often to check it, and what a meaningful change looks like versus normal noise. Use only what is in the data I gave you. If something is not in it, say so rather than filling it in.
AI cannot forecast lumber prices. Use it to organize supplied evidence and create monitoring scenarios—not to invent the next price.

Build a tiny HTML tool

Choose one annoying task or follow the Timber Sale Tracker example.

Build
Build a single-file HTML tool to help with this problem: [describe problem]. It should have inputs, output, review warning, no external dependencies, and a print button. Ask up to five questions before coding if needed.
Refine
Two changes only. [First change.] [Second change.] Leave everything else exactly as it is.
Open the Timber Sale Tracker example
A prototype is not a production system. Ownership, testing, access, persistence, maintenance, and review still have to be decided.

Four patterns worth keeping.

These cover most of the work from this session.

Specificity

[What it is] — [style and era], [materials and colors], [distinguishing features], [surroundings], [location], [season and time], [light], [vantage point], [format].

Intention

[Do this thing] for [who will use it], because [what happens next]. Keep it [length and format]. Leave out [anything unsafe or unnecessary].

Data

Build [what] with columns for [fields]. Add [calculation] and subtotals by [grouping]. Write the formulas in the cells, not just the answers. Flag assumptions and questionable rows.

Teach me back

Looking back at everything we just worked through, how could I have gotten this from a single prompt? Write me that prompt, then tell me which parts mattered most.

Before any output leaves your screen.

The prompt does not replace accountability.

  • Confirm the audience and purpose.
  • Check every name, number, date, formula, source, and commitment.
  • Remove unsupported claims and anything the tool invented.
  • Use approved tools and approved data only.
  • If a number cannot be traced to a source, do not use it.
  • A person signs off.
